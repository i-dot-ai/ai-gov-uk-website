Develop a business case summary

Develop a concise summary for senior decision-makers, based on the Five Case Model.

Verified
The approach in this prompt has been verified by the Government Project Delivery Professionals. You must still check the results from this prompt for accuracy, as AI can make mistakes.

Email ai-knowledge-hub@dsit.gov.uk to share feedback about this prompt.

What this prompt does

This prompt is suitable for any AI assistant (for example, Microsoft Copilot or Gemini). The prompt will:

  • follow the structure of the Five Case Model
  • aligns with HM Treasury's Green Book guidance
  • cover all five cases: Strategic, Economic, Commercial, Financial, and Management

Before you start

Before you use this prompt, replace text in square brackets: *[DESCRIBE THE PROJECT], for example [building a new university] *[PROJECT STAGE], for example [SOC or FBC] *[AUDIENCE TYPE], for example [ministerial briefing or delivery partner]

Do not upload any personal or official information to free tools, or any tool not advised by your department. If you are unsure, check with your digital team.

The prompt

              //Goal

You are an experienced Business Case Manager working on a programme. Create a summary business case using the Five Case Model for the following programme [DESCRIBE THE PROJECT].

//Context

This business case summary is intended for senior decision-makers and must align with the sources outlined below.

//Source

Source:
* HM Treasury’s Green Book guidance (https://assets.publishing.service.gov.uk/media/6645c709bd01f5ed32793cbc/Green_Book_2022__updated_links_.pdf)
* Guide to developing the project business case (https://assets.publishing.service.gov.uk/media/66449468ae748c43d3793bb8/Project_Business_Case_2018.pdf)
* Structure must follow the Five Case Model

//Expected output

We are in [PROJECT STAGE].

Expected Output:
* If any programme details are missing, ask for clarification before proceeding.
* If assumptions are necessary, clearly state them and flag any speculative or unverifiable content.
* Use a professional, executive summary tone suitable for non-technical stakeholders.

Structure the output using the Five Case Model:
1. Provide an overview of the project/programme and what it aims to achieve. Including programme title, purpose, scope, target completion date, and estimated overall programme cost.
2. Strategic Case: Explain the case for change and how the project aligns with strategic priorities. 
3. Economic Case: Outline the options considered, the preferred option, value for money, and wider economic benefits. 
4. Commercial Case: Describe how the preferred option supports a viable procurement strategy and a well-structured deal with providers. 
5. Financial Case: Summarise affordability, funding sources, financial implications, and stakeholder/customer support. 
6. Management Case: Provide a high-level delivery plan, governance structure, risk management approach, and assurance strategy. If information on top risks is available, include this with proposed mitigations.

//Output 

* Present the outputs in clear bullet points for [AUDIENCE TYPE]
* Include confidence ratings for key claims and cite relevant sources where applicable
* Avoid speculative or unverified claims
