AI assistants use written, visual or audio information and instructions (a prompt) and create the most likely response. The clearer the prompt, the better the result. Find out how to use AI at work.

AI can produce content that sounds correct but is wrong.

Always check AI output for accuracy, as it can give false information, outdated details or biased outputs.

The best way to learn is by experimenting, for example, you can:

  • draft and test prompts yourself
  • use AI to improve your prompt-writing skills
  • explore our prompt library for inspiration

Write simple prompts for everyday tasks

Most everyday tasks need a simple, clear instruction, for example, to: draft an email or letter

  • summarise a short document
  • create a basic list
  • get a quick answer to a question

Tell a prompt what you want

To do this: say exactly what you need use direct language start with action words like 'write', 'analyse', 'create' or 'list'.

❌ Bad prompt: Create a dashboard

✔️Good prompt: Create a dashboard showing monthly sales, customer numbers and product performance

Tell a prompt who it’s for

Be detailed about your audience and their background so it will use the right tone and level of detail, for example:

  • senior managers
  • civil servants who have no technical background in this area
  • engaged university students

Specify the format

Tell the prompt how you want to see the information and the style it should have, for example:

  • word count (under 200 words)
  • format (email, bullet points, table)
  • tone (formal, conversational)

❌ Bad prompt: Explain the new policy

✔️Good prompt: Explain the new holiday policy in an email to staff. Keep it under 150 words and use a friendly tone

Give relevant background information

Include details the AI needs to give you a useful and relevant answer.

❌ Bad prompt: Create a meal plan

✔️Good prompt: Create a 7-day meal plan for someone managing diabetes. Include 1,800 calories daily. Show breakfast, lunch, dinner and nutritional information

Ask AI to improve your prompts

Before you run a prompt, you can ask AI to help you make it better. For example:

  • ‘Before you answer, review my prompt and suggest 3 ways to make it clearer’
  • ‘Before you answer, ask me 3 clarifying questions to check you've understood this prompt correctly’
  • ‘I'm using this prompt with [AI tool name]. Review it and suggest ways to improve it for better results’

Write detailed prompts for complex tasks

Some tasks need more information to get good results. These are usually tasks that involve multiple steps, need specific expertise, or produce longer outputs.

Use detailed prompts when you need to:

  • analyse data and draw conclusions
  • create reports or strategies
  • work with multiple documents
  • produce detailed plans

You can structure detailed prompts using frameworks to help you remember what information to include.

Use the RACE framework for focused tasks

RACE works well for tasks that need one clear output, like identifying risks or drafting summaries. Write you prompt with these sections:

  • Role - tell the AI what perspective to take (for example, ‘You are a risk manager on a construction project’)
  • Action - say what you want it to do (for example, ‘Identify the top 5 delivery risks’)
  • Context - give necessary background (for example, ‘For a £200 million road upgrade in a busy city centre’)
  • Expected output - specify the format (for example, ‘Under 150 words, use bullet points, write in plain English’)

Use the GCSE framework when you have source documents

GCSE helps when you're working with existing documents or data, on tasks that need accuracy and compliance. Write you prompt with these sections:

  • Goal - say what you want AI to do (for example, ‘Create a project schedule’)
  • Context - provide background information (for example, ‘The project is a website redesign for 50,000 users. We have 3 developers and a 6-month deadline’)
  • Source material - list documents or data to use (for example, ‘Use the project brief dated March 2025 and the technical requirements document’)
  • Expected output - describe the format (for example, ‘Show as a table with dates, milestones, owners and dependencies’)

Use the CRIT framework for strategic or creative tasks

CRIT works well for tasks where structure and tone matter, like drafting frameworks or evaluation criteria. Write your prompt with these sections:

  • Context - provide background (for example, ‘This is for a public infrastructure project’) Role - assign a relevant role (for example, ‘You are a procurement advisor’)
  • Intent - define your goal (for example, ‘Draft a sustainability scoring matrix’)
  • Type of output - specify format and tone (for example, ‘Table with criteria, weights and evaluation notes’)

Break down complex tasks into steps

Some tasks are too complicated for one prompt. Split them into separate steps, where each prompt builds on the previous output. Use this approach when you need to:

  • produce a major report or strategy
  • do detailed analysis that needs validation
  • work through multiple stages of a process

Each step should have one clear purpose. Use the output from one step as input for the next, for example:

  1. 'Summarise this 50-page report. Focus on key findings and recommendations'.
  2. 'Review the summary above for accuracy and completeness. List any important details I missed'.
  3. ‘Improve the summary using this feedback’: [paste feedback from 2]
